# vertx-framework

这个框架在不断的优化中，很多的idea都是在某些个瞬间产生的，这里记录一些关于此框架一些优化的记录。

* 2018-06-20 链路监控意识形成
* 2018-06-22 vertx-framework 项目初始提交
* 2018-07-03 HTTP监听、管理端、监控端都可以作为插件，形成微核心意识，很多的基础功能都是都通插件完成
* 2019-02-20 第一版注解+Bean形成，但问题比较多，开发复杂，不便于理解。
* 2019-03-01 形成新的体系结构
* 2019-03-16 初版形成，并能够跑通第一个服务链
* 2019-03-20 一直都有一个想法，就是既可以便于开发复杂的API服务，也可以开发常规的Web服务，今天想到可以在链上做文章。
* 2019-03-23 服务参数支持JsonObject和String两种类型的注入
* 2019-03-24 ChainBean定义一个标识，来区分是普通链还是特殊链。如果是普通链，在监听中就按照原来的方式执行，如果是特殊链，则直接将ChainBean中的Handler<RoutingContext>属性值给路由。
* 2019-03-25 ChainBean中不需要区分是普通链还是特殊链，不管是普通链还是特殊链，实际上都是转成Handler<RoutingContext>的形式。所以可在ChainBean中给Handler<RoutingContext> handle 赋上一个默认值，这个默认值就是执行当前实例中service，如果开发者重新给这个属性赋值，那么也就表示是一个特殊链了。
* 2019-03-26 ↑面的想法虽好，但不好实现￣□￣｜｜
